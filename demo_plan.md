# 项目展示规划 (预计 3-4 分钟)

这份规划旨在全面展示文件管理系统的核心功能、全新的"极客暗黑"界面风格和内核级命令行操作。

**注意：本系统模拟老式文件系统，文件名限制严苛（目录名最多3字符，文件名最多3字符，扩展名最多2字符）。**

---

## 第一部分：图形化界面 (GUI) 展示 (约 2 分钟)

**目标**：展示新 UI 风格、文件操作、磁盘可视化和系统信息。

### 1. 启动与界面介绍 (20秒)
-   **展示桌面**：启动系统，展示 **"极客暗黑风"** 桌面。
-   **特色说明**：
    -   左下角任务栏布局
    -   直角窗口设计（Anti-Round）
    -   霓虹绿 (`#00E676`) 强调色
-   **操作**：鼠标悬停在任务栏图标上，展示高亮悬停效果。

### 2. 文件管理器操作 (40秒)
-   **打开文件管理器**
-   **新建目录**：
    -   右键 -> 新建文件夹 -> 命名为 `PRJ` (3字符限制演示)
    -   双击进入 `PRJ`
-   **新建文件**：
    -   右键 -> 新建文件 -> 命名为 `R.ME`
    -   **解说**："文件名3字符，扩展名2字符，符合 FAT 文件系统规范。"
-   **文件属性查看**：
    -   右键 `R.ME` -> 属性
    -   展示属性窗口（文件大小、起始盘块号、属性标志等）

### 3. 磁盘使用情况可视化 (30秒)
-   **操作**：点击任务栏"磁盘使用情况"（或从桌面打开）
-   **解说**：
    -   "绿色块代表已分配的文件数据块"
    -   "深灰色是空闲块"
    -   "红色表示坏块（可通过磁盘修复工具模拟和修复）"
-   **演示**："观察刚才创建的 `PRJ` 和 `R.ME` 占用了哪些盘块。"

### 4. 团队分工信息 (30秒)
-   **操作**：回到桌面，双击 **"分工明细"** 图标
-   **展示**：
    -   窗口标题已从 "关于我们" 更改为 **"分工明细"**
    -   显示团队成员及各自负责的模块

---

## 第二部分：命令行 (CLI) 深度展示 (约 1.5-2 分钟)

**目标**：展示系统内核级指令的完整性、文件读写流程和权限控制。

### 1. 基础导航与文件创建 (30秒)
-   **打开命令行窗口**
-   **输入**：`/h`
    -   *效果*：显示所有可用命令列表
-   **输入**：`/pwd`
    -   *效果*：显示当前工作目录（默认 `/`）
-   **输入**：`/dir /`
    -   *效果*：列出根目录文件，应该能看到刚才 GUI 创建的 `PRJ`
-   **输入**：`/cd /PRJ`
    -   *效果*：切换到 `PRJ` 目录
-   **输入**：`/create_file DAT.TX 6`
    -   *解说*："创建普通文件（属性6：系统+隐藏）。"

### 2. 文件写入流程 (40秒)
-   **输入**：`/write_buffer Hello_Kernel`
    -   *解说*："将数据写入缓冲区，这是写入的第一步。"
-   **输入**：`/open_file DAT.TX rw`
    -   *效果*：文件以读写模式打开
-   **输入**：`/write_file DAT.TX`
    -   *解说*："将缓冲区数据刷入磁盘，完成持久化。"
-   **输入**：`/close_file DAT.TX`
    -   *解说*："关闭文件句柄，释放系统资源。"

### 3. 文件内容验证 (20秒)
-   **输入**：`/typefile DAT.TX`
    -   *效果*：显示文件属性和完整内容 "Hello_Kernel"
    -   *解说*："typefile 命令可以直接查看文件的原始内容。"

### 4. 权限控制演示 (Optional, 20秒)
-   **输入**：`/change DAT.TX 7`
    -   *解说*："将文件属性改为7（只读+系统+隐藏）。"
-   **输入**：`/open_file DAT.TX rw`
    -   *效果*：提示 "不能以写方式打开只读文件"
    -   *解说*："系统严格的权限控制机制。"

### 5. 清理与验证 (Optional, 10秒)
-   **输入**：`/delete_file DAT.TX`
    -   *效果*：可能提示文件只读或成功删除
-   **输入**：`/dir /PRJ`
    -   *验证*：查看目录是否清空

## 演示技巧建议

1.  **语速控制**：保持流畅但不急促，重要操作适当停顿。
2.  **高亮重点**：特别强调以下亮点：
    -   极客暗黑主题的视觉冲击
    -   FAT 表可视化的直观性
    -   CLI 命令的专业性和系统级控制
3.  **预先准备**：提前格式化磁盘或准备干净环境。
4.  **时间弹性**：如果时间紧张，可以省略 Optional 部分；如果时间充裕，可以增加磁盘修复演示。

常用属性组合示例
属性可以组合使用（相加），例如：

0 (普通文件): 无任何特殊属性
1 (只读): 0x01
2 (隐藏): 0x02
4 (系统): 0x04
6 (系统+隐藏): 0x04 + 0x02 = 6
7 (只读+系统+隐藏): 0x01 + 0x02 + 0x04 = 7
8 (目录): 0x08
12 (普通目录): 0x08 + 0x04 = 12 (目录 + 系统标志)
